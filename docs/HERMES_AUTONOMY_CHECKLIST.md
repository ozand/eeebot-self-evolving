# Hermes Autonomy Checklist

Use this checklist before ending any progress/status reply.

Current time included:
- [ ] current time was obtained from a tool and included in the reply

Status structure:
- [ ] I included `What I am doing now:`
- [ ] if anything is delegated, I included `What is delegated:`

Execution truth:
- [ ] every claimed action was actually performed in the same response/session
- [ ] any issue marked in-progress has real execution or delegation already started

No premature stopping:
- [ ] I am not stopping while another open bounded issue exists and no blocker is present
- [ ] I am not ending on a summary or handoff sentence
- [ ] I am not using future-step phrasing instead of continuing work
- [ ] I am not using conditional continuation language such as `if you want` or `I can continue`

Issue discipline:
- [ ] GitHub Issue is the source of task truth
- [ ] lifecycle labels/comments were updated if work advanced
- [ ] rollout/proof was linked back to the issue when applicable

Failure discipline:
- [ ] if a bounded attempt failed, I repaired or rolled back to a green baseline
- [ ] I did not leave half-broken source state while claiming progress
